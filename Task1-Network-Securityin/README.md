## 📌 Objective

The objective of this task was to understand the fundamental concepts of network security and identify common network threats. This task also involved implementing basic security measures and analyzing network traffic using Wireshark.

---

## 🛠 Tools Used

* **Wireshark** – For capturing and analyzing network traffic
* **Kali Linux** – Security testing environment
* **Windows Defender Firewall** – Basic network protection

---

## ⚠ Common Network Threats Studied

During this task, several common network threats were researched and understood:

* **Virus** – Malicious software that infects files and spreads when executed.
* **Worm** – Self-replicating malware that spreads across networks automatically.
* **Trojan Horse** – Malicious software disguised as legitimate programs.
* **Phishing Attack** – Social engineering attack used to steal sensitive information like passwords or banking details.

---

## 🔒 Security Measures Implemented

The following basic security configurations were implemented to protect the system and network:

* Enabled **Windows Defender Firewall** to block unauthorized access.
* Verified **network security settings** to ensure safe connectivity.
* Ensured **Wi-Fi encryption (WPA2/WPA3)** was active for secure communication.

These measures help protect systems from unauthorized access and network-based attacks.

## 📊 Key Findings

* DNS requests are responsible for translating domain names into IP addresses.
* HTTP traffic represents communication between client systems and web servers.
* Network monitoring tools like Wireshark help security professionals detect suspicious or unusual network activity.

---

## 🛡 Security Best Practices

To protect networks from cyber threats, the following best practices should be followed:

* Enable firewalls to block unauthorized access.
* Use strong and unique passwords.
* Use encrypted wireless networks (WPA2/WPA3).
* Regularly monitor network traffic.
* Keep systems and software updated.
